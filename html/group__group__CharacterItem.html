[InventoryRewrite] Item is broken. Consider the following approaches:
  • Items should contain their own name
  • Items have their own "data"
  • Items match only if all "data" matches
    • What about count?
  • Missing "familiarity" by the user of the Item
    • Could this be handled by Custom Data?
  • Add Policy controller support
    • Policy Stacking
      • Set Default Stacking value
      • Data Key: "Yars.ItemPolicy.Stacking.MaxCount"
    • Policy Weight
    • Policy Packing
    • Policy uses data "Yars.ItemPolicy.Stacking.KEY"

Items allow a Character to use their Skills. A Character must have the Skill to use Item as well as having that Item in their possession. The outcome of using an Item is in Affect is created. The Affect can then be applied to a Character.
